# Hybrid Smart Attendance System - Project Workflow

Use this guide to structure your presentation. Each section represents a key screen or action in the application where you can include a screenshot.

## 1. Introduction / Landing Page
*   **Screen:** The main landing page with options for "Student Login" and "Teacher Login".
*   **Action:** User chooses their role.

---

## 2. Student Workflow

### A. Student Registration
*   **Screen:** Student Registration Form (`/student/register`).
*   **Details:** Student enters Name, Roll Number, Branch, Semester, Password, and uploads a Profile Photo.
*   **Key Feature:** Secure password hashing and photo upload.

### B. Student Login
*   **Screen:** Student Login Form (`/student/login`).
*   **Details:** Student logs in using Roll Number and Password.
*   **Key Feature:** JWT Token authentication.

### C. Student Dashboard
*   **Screen:** Student Dashboard (`/student/dashboard`).
*   **Details:** Shows student profile, quick stats, and menu options.
*   **Key Feature:** Clean, responsive UI with glassmorphism design.

### D. Biometric Registration (WebAuthn)
*   **Screen:** "Register Fingerprint" section on Dashboard.
*   **Action:** Student clicks "Register Fingerprint" and uses their device's biometric sensor (TouchID, Windows Hello, Android Fingerprint).
*   **Key Feature:** Passwordless security using WebAuthn/FIDO2.

### E. Marking Attendance (QR Scan)
*   **Screen:** QR Scanner Page (`/student/scan-qr`).
*   **Action:** Student scans the QR code generated by the teacher.
*   **Process:**
    1.  Scan QR Code.
    2.  **Biometric Verification:** System prompts for fingerprint/face unlock to confirm identity.
    3.  **Location Check:** (Optional) Verifies student is within range.
    4.  **Success:** "Attendance Marked Successfully" message.

### F. View History
*   **Screen:** Attendance History Page (`/student/history`).
*   **Details:** List of all attended classes with dates and status.

---

## 3. Teacher Workflow

### A. Teacher Registration & Login
*   **Screen:** Teacher Registration/Login Forms (`/teacher/register`, `/teacher/login`).
*   **Details:** Teacher signs up with Email and Subject.

### B. Teacher Dashboard
*   **Screen:** Teacher Dashboard (`/teacher/dashboard`).
*   **Details:** Overview of recent sessions and options to manage attendance.

### C. Generate QR Code
*   **Screen:** Generate QR Page (`/teacher/generate-qr`).
*   **Action:** Teacher selects Subject, Branch, and Semester.
*   **Key Feature:**
    *   **Dynamic QR:** The QR code updates every few seconds to prevent sharing.
    *   **Geofencing:** Teacher can set a valid radius (e.g., 50 meters) for attendance.

### D. View Attendance & Reports
*   **Screen:** View Attendance Page (`/teacher/attendance/:sessionId`).
*   **Details:** Real-time list of students who have marked attendance.
*   **Action:** Teacher can manually mark/unmark students or export the list.

### E. Notifications
*   **Screen:** Notifications Page.
*   **Action:** Teacher sends announcements to specific batches (e.g., "Class cancelled").
